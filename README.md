# UE5 物联网学习进程总结

## 项目概述

这个仓库用于总结我在 UE5 与物联网结合方向上的学习过程。目前的学习内容主要围绕设备数据采集、虚拟机网络配置、后端接口搭建，以及 UE5 前端界面展示几个部分展开。

现阶段已经完成了基础链路的打通：边缘网关与温湿度传感器产生的数据可以通过 MQTT 传输到 Rocky Linux 虚拟机中，同时 UE5 也能够通过接口获取历史数据并完成可视化展示。

## 当前学习进展

### 1. 边缘网关与温湿度传感器通过 MQTT 连接虚拟机

目前已经学习并实践了边缘网关和温湿度传感器的数据采集流程，并通过 MQTT 协议将设备数据传输到 Rocky Linux 虚拟机。

在这个过程中，我掌握的一项关键新知识是：

- 在校园网内网环境下，如何同时实现虚拟机联网和桥联模式接入设备。
- 具体做法是为虚拟机配置两张网卡。
- 一张网卡用于虚拟机访问外网或校园网，保证系统联网和服务部署。
- 另一张网卡用于桥联设备网络，使虚拟机能够直接接收来自边缘网关和传感器的数据。

这部分学习让我对虚拟机网络配置、实际设备接入方式，以及 MQTT 在物联网数据传输中的作用有了更直观的理解。

### 2. UE5 的 2D UI 界面配置与框架搭建

在 UE5 部分，我学习了 2D UI 界面的基础配置方法，以及整体界面框架的搭建思路。

目前主要掌握了以下内容：

- UE5 中 2D UI 的基础配置流程。
- UI 界面结构的组织方式。
- 项目中基础界面框架的搭建方法。

这部分内容为后续将物联网数据接入 UE5 并进行可视化展示打下了基础。

### 3. Node-RED API 搭建与历史数据可视化

目前还完成了 Node-RED 的基础配置，并使用 Node-RED 创建了 API 接口。

在这一部分中，已经实现的能力包括：

- 通过 Node-RED 提供固定 API。
- UE5 通过 HTTP 请求访问该 API。
- API 从 Rocky Linux 虚拟机中的 MySQL 数据库读取历史数据。
- UE5 在 UI 界面中展示历史数据曲线。

这意味着当前系统已经具备了从数据库读取历史记录并在 UE5 前端进行可视化的基本能力，为后续做实时监控界面和更完整的数据看板提供了基础。

## 当前阶段成果

截至目前，已经初步打通了以下流程：

`传感器/边缘网关 -> MQTT -> Rocky Linux 虚拟机 -> MySQL -> Node-RED API -> UE5 UI 界面`

这说明整个物联网数据链路已经具备了基础雏形，不仅实现了设备数据接入，也实现了历史数据查询与前端展示。
