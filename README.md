# UE5 物联网学习进程总结

## 概述

这个仓库用于总结在 UE5 与物联网结合方向上的学习过程。

## 当前学习进展

### 1. 边缘网关与温湿度传感器通过 MQTT 连接虚拟机

目前已经学习并实践了边缘网关和温湿度传感器的数据采集流程，并通过 MQTT 协议将设备数据传输到 Rocky Linux 虚拟机。

- 在校园网内网环境下，如何同时实现虚拟机联网和桥联模式接入设备。
- 具体做法是为虚拟机配置两张网卡。
- 一张网卡用于虚拟机访问外网或校园网，保证系统联网和服务部署。
- 另一张网卡用于桥联设备网络，使虚拟机能够直接接收来自边缘网关和传感器的数据。

### 2. UE5 的 2D UI 界面配置与框架搭建

在 UE5 部分，我学习了 2D UI 界面的基础配置方法，以及整体界面框架的搭建。

目前主要掌握了以下内容：

- UE5 中 2D UI 的基础配置流程。
- UI 界面结构的组织方式。
- 项目中基础界面框架的搭建方法。

### 3. Node-RED API 搭建与历史数据可视化

目前还完成了 Node-RED 的基础配置，并使用 Node-RED 创建了 API 接口。

在这一部分中，已经实现的能力包括：

- 通过 Node-RED 提供固定 API。
- UE5 通过 HTTP 请求访问该 API。
- API 从 Rocky Linux 虚拟机中的 MySQL 数据库读取历史数据。
- UE5 在 UI 界面中展示历史数据曲线。


## 当前阶段成果

截至目前，已经初步打通了以下流程：

`传感器/边缘网关 -> MQTT -> Rocky Linux 虚拟机 -> MySQL -> Node-RED API -> UE5 UI 界面`


## 阶段总结

 - 服务器：我们是通过使用虚拟机来部署mqtt_server和mySQL数据库的，它其实就是在模拟工控机的角色。
 - UI界面：想要做出真正精美的UI界面是需要PS功底的，目前看到的界面是通过gpt+PS截取的方式，所以后续需要了解一下PS的使用
 - UE5代码：目前课程中的代码都可以看懂，但单独编写还需要再练习。
 - Node-RED：Node-RED在这里发挥的作用主要是提供API，将mysql存储的数据发送给UE5前端。
   我们不使用UE5直接接收mysql数据的原因：
     第一，我们在历史数据面板中需要的是每小时平均温湿度数据，我们可以在Node-RED中处理完后再传给UE5，减小对UE5的渲染速度和质量的影响。
     第二，我们可以通过API的date参数实现数据查询功能。
